# Running a Shinzo Host

This page covers installing a Shinzo Host locally from source or on a virtual machine with Docker. A Host turns indexed blockchain data into structured [Views](/views/overview) and produces [Attestation Records](/reference/components/host-client) that help secure the network. To complete a Host setup, you must also [register it with the Shinzo Network](/hosts/register).

## Hardware recommendations

| Component | Minimum | Recommended |
| --- | --- | --- |
| CPU | 8 vCPUs | 16 vCPUs |
| Memory (RAM) | 16 GB | 32–64 GB |
| Storage | 3 TB NVMe | 4+ TB NVMe |
| OS | Ubuntu 24.04 | Ubuntu 26.04 |

## Local deployment

Run the Host Client directly on your machine for development and testing. For production or devnet participation, use the [VM deployment](#vm-deployment) instead.

### Prerequisites

- Make
- Git
- A browser wallet (such as MetaMask). The wallet does not need to hold any funds.

### Steps

1. Clone the repository.

    ```shell
    git clone https://github.com/shinzonetwork/shinzo-host-client.git
    cd shinzo-host-client
    ```

1. Set the keyring secret:

    ```shell
    export DEFRA_KEYRING_SECRET=<make-a-password>
    ```

1. Build and run the binary:

    ```shell
    make build
    make start
    ```

:::info
The included `config.yaml` works for most local development. You typically only need to change peer settings or storage paths for advanced setups. See the [Configuration reference](#configuration-reference) for details.
:::

### Running an Indexer and Host on the same machine

If you are running your own Indexer, connect your Host to it by configuring `p2p.bootstrap_peers`. To get the required Peer ID, query the Indexer's registration endpoint:

```shell
curl http://localhost:8080/registration
```

From this information, assemble your peer connection info: `/ip4/<your-ip-here>/tcp/9171/p2p/<your-PeerID-here>`. Replace the default peer in `p2p.bootstrap_peers` with your Indexer peer.

When the Indexer and Host run on the same machine, apply the following changes to the [Host's `config.yaml`](https://github.com/shinzonetwork/shinzo-host-client/blob/main/config/config.yaml) to avoid port collisions.

The Indexer is likely already using port `9181`, so update the `defradb.url` field:

```yaml
url: "localhost:9182"
```

Also update the [P2P settings](https://github.com/shinzonetwork/shinzo-host-client/blob/main/config/config.yaml#L4) to use localhost and a different port so the Host doesn't clash with the Indexer:

```yaml
bootstrap_peers:
  - '/ip4/127.0.0.1/tcp/9171/p2p/<PeerID>'
listen_addr: "/ip4/0.0.0.0/tcp/9172"
```

### (Optional) Enable the GraphQL Playground

The Host ships with an optional web-based GraphQL Playground for querying the embedded DefraDB instance.

```shell
make build-playground
make start
```

This runs the Host and exposes a Playground GUI. Check the output logs for the address:

```plaintext
🧪 GraphQL Playground available at ...
```

The Playground lets you run GraphQL queries against primitive data and any Views your Host is serving. Try this query:

```graphql
query GetLatestLogs {
  Ethereum__Mainnet__Log(
    order: { blockNumber: DESC }
    limit: 10
  ) {
    address
    topics
    data
    blockNumber
    blockHash
    transactionHash
    transactionIndex
    logIndex
    removed
  }
}
```

More query examples are available on the [Examples](/hosts/examples) page.

## VM deployment

This is the recommended approach for production and devnet participation. It uses Docker, docker-compose, and Nginx on a virtual machine.

### Prerequisites

- A virtual machine running Ubuntu 24.04 or later.
- Docker, including the `docker compose` plugin.
- Ports `443` (HTTPS) and `9171` (P2P) open in your firewall or security group.

### Steps

1. Create the data directory. The container runs as UID `1001`, so the directories must be owned by that user.

    ```shell
    sudo mkdir -p ~/data/defradb ~/data/lens
    sudo chown -R 1001:1001 ~/data/defradb ~/data/lens
    ```

1. Generate the SSL certificates.

    ```shell
    set -e &&
    sudo mkdir -p ~/ssl &&
    sudo openssl genrsa -out ~/ssl/nginx.key 2048 &&
    sudo openssl req -new -key ~/ssl/nginx.key -out /tmp/nginx.csr -subj "/C=US/ST=State/L=City/O=Shinzo/OU=Host Client/CN=shinzo.network" &&
    sudo openssl x509 -req -days 365 -in /tmp/nginx.csr -signkey ~/ssl/nginx.key -out ~/ssl/nginx.crt &&
    sudo rm /tmp/nginx.csr
    ```

1. Write the configuration file. Create `~/config.yaml`. The only value you must change is `defradb.keyring_secret`. See the [Configuration reference](#configuration-reference) for the other fields.

    ```yaml
    defradb:
      url: "localhost:9181"
      keyring_secret: "<YOUR_SECRET>"        # Required, change this
      p2p:
        enabled: true
        bootstrap_peers:
          - '/ip4/34.63.13.57/tcp/9171/p2p/12D3KooWMYhYNBo4zAi9j7TpyGQJBSvbwSSNkgsMrLs6vHUnFUzY'
        listen_addr: "/ip4/0.0.0.0/tcp/9171"
        enable_auto_reconnect: true
      store:
        path: "./.defra"
    shinzo:
      hub_base_url: rpc.devnet.shinzo.network:26657
      minimum_attestations: 1
    logger:
      development: false
      level: "error"
    host:
      lens_registry_path: "./.defra/lens"
      health_server_port: 8080
    ```

1. Write the Nginx config. Create `~/nginx.conf`. Nginx terminates TLS using the certificates generated above and proxies to the Host container.

    ```nginx
    events { worker_connections 1024; }

    http {
      map $http_origin $cors_origin {
        default "";
        "https://explorer.shinzo.network" $http_origin;
      }

      server {
        listen 443 ssl;
        server_name _;

        ssl_certificate     /etc/nginx/ssl/nginx.crt;
        ssl_certificate_key /etc/nginx/ssl/nginx.key;
        ssl_protocols       TLSv1.2 TLSv1.3;

        add_header 'Access-Control-Allow-Origin' $cors_origin always;
        add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE, OPTIONS' always;
        add_header 'Access-Control-Allow-Headers' 'Authorization, Content-Type, Accept, Origin' always;
        add_header 'Access-Control-Max-Age' 3600 always;
        add_header 'Vary' 'Origin' always;

        location / {
          if ($request_method = OPTIONS) { return 204; }
          proxy_pass http://shinzo-host:9181;
          proxy_set_header Host $host;
        }

        location = /metrics {
          if ($request_method = OPTIONS) { return 204; }
          proxy_pass http://shinzo-host:8080/metrics;
          proxy_set_header Host $host;
        }

        location = /api/v0/graphql {
          if ($request_method = OPTIONS) { return 204; }
          proxy_pass http://shinzo-host:9181/api/v0/graphql;
          proxy_set_header Host $host;
        }
      }
    }
    ```

1. Write the docker-compose file. Create `~/docker-compose.yml`.

    ```yaml
    networks:
      shinzo-net:
        driver: bridge

    services:
      shinzo-host:
        image: ghcr.io/shinzonetwork/shinzo-host-client:standard
        user: "1001:1001"
        mem_limit: 16g
        mem_reservation: 13g
        restart: unless-stopped
        container_name: shinzo-host
        networks:
          - shinzo-net
        ports:
          - "8080:8080"   # Health, metrics, and registration endpoint
          - "9181:9181"   # DefraDB API (proxied by Nginx in production)
          - "9171:9171"   # P2P networking
        volumes:
          - ~/data/defradb:/app/.defra/data
          - ~/data/lens:/app/.lens
          - ~/config.yaml:/app/config.yaml:ro
        environment:
          - DEFRA_URL=0.0.0.0:9181
          - GOMEMLIMIT=14GiB
          - LOG_LEVEL=error
          - LOG_SOURCE=false
          - LOG_STACKTRACE=false
        healthcheck:
          test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:8080/metrics"]
          interval: 15s
          timeout: 30s
          retries: 10
          start_period: 120s

      nginx:
        image: nginx:alpine
        ports:
          - "443:443"
        volumes:
          - ~/nginx.conf:/etc/nginx/nginx.conf:ro
          - ~/ssl:/etc/nginx/ssl:ro
        depends_on:
          - shinzo-host
        networks:
          - shinzo-net
        restart: unless-stopped
    ```

1. Start the Host.

    ```shell
    docker-compose up -d
    ```

### Monitoring

The metrics endpoint is reachable through Nginx at:

```shell
https://<VM_IP>/metrics
```

The container health check polls `http://localhost:8080/metrics` inside the container every 15 seconds. To check the container status and logs from the VM:

```shell
docker ps
docker logs shinzo-host
```

### Docker image

The multi-stage Dockerfile builds the Host binary (Go 1.25) along with the Wasmtime and Wasmer WASM runtimes. The production image is based on Ubuntu 24.04 and runs as a non-root `shinzo` user. Pre-built images are published to:

```shell
ghcr.io/shinzonetwork/shinzo-host-client:standard
```

## Configuration reference

The Host Client reads from [`config.yaml`](https://github.com/shinzonetwork/shinzo-host-client/blob/main/config/config.yaml). The defaults work for most local development; the only field you must set is `defradb.keyring_secret`.

| Field | Description |
| --- | --- |
| `defradb.url` | API endpoint of your local DefraDB node. Defaults work for most setups. |
| `defradb.keyring_secret` | Secret used to generate your private keys. **Required.** Can be set with the `DEFRA_KEYRING_SECRET` environment variable instead. |
| `p2p.bootstrap_peers` | Indexer peers for receiving indexed data. Defaults include a reliable bootstrap peer. |
| `p2p.listen_addr` | The P2P listen address. The default is suitable for local runs; override it when containerizing. |
| `store.path` | Directory where local DefraDB data is stored. |
| `shinzo.hub_base_url` | ShinzoHub node URL. Only change if connecting to a different node. |
| `shinzo.minimum_attestations` | Minimum number of independent Indexers required before data is served. |
| `logger.development` | Set to `false` for production. |
| `host.lens_registry_path` | Where received WASM lens files are stored. |
| `host.health_server_port` | Port for the health and metrics endpoint. |

## Exposed ports

The following ports are used by the Host. In the VM deployment, only `443` and `9171` need to be open in your firewall; the rest are bound to the VM and reached through Nginx or an SSH tunnel.

| Port | Service | Public in VM deployment |
| --- | --- | --- |
| `443` | HTTPS endpoint served by Nginx (proxies `/`, `/metrics`, `/api/v0/graphql` to the Host). | Yes |
| `9171` | DefraDB P2P. | Yes |
| `8080` | Health and metrics endpoint (`/metrics`) and registration (`/registration-app`). Reach via SSH tunnel for registration. | No |
| `9181` | DefraDB GraphQL API. Proxied by Nginx in production. | No |
| `9182` | Optional GraphQL Playground (local dev only, requires `make build-playground`). | No |

## Registration

Once your Host is running, register it with the Shinzo Network so it can replicate data and earn rewards. See the [Registration guide](/hosts/register) for the GUI and CLI flows.

## Troubleshooting

### Permission denied on `.defra/keys`

The data directories are owned by root but the container runs as UID 1001. Stop the container, fix the ownership, then start it again:

```shell
docker-compose -f ~/docker-compose.yml stop
chown -R 1001:1001 ~/data/defradb ~/data/lens
docker-compose -f ~/docker-compose.yml start
```

### Failed to load existing DefraDB identity

`defradb.keyring_secret` has changed since the first run. Restore the original value in your config and restart.

### Port already in use

The Indexer and Host both use DefraDB and P2P ports. If you run them on the same machine, follow [Running an Indexer and Host on the same machine](#running-an-indexer-and-host-on-the-same-machine) to move the Host onto different ports.
